Nanodegree Engenheiro de Machine Learning¶

  • Projeto Final: Avaliao de modelos de predio de pacientes de alto custo no Sistema nico de Sade.
  • Base de dados: Datasus
  • Perodo de avaliao: Janeiro de 2016 a Dezembro de 2017
  • Objetivo final: Estimativas indicam que 10% dos pacientes portadores de doenas crnicas representam 2/3 dos custos totais com sade no Brasil. Desta forma, o objetivo deste projeto ser predizer com acurcia quais indivduos possuem maior risco de ser um paciente de alto custo (H_COST).

  • VAL_PED1AC:contnuo. Valor de pediatria - primeira consulta

  • VAL_UTI:contnuo. Valor referente aos gastos em UTI

Sero inseridas duas variveis, conforme abaixo:

  • PERCENTIL95: contnuo.

    • Valor calculado do percentil de 95% dos custos mdicos totais.

  • H_COST: DISCRETO.

    • 1 - se paciente teve custos mdicos superiores ao percentil de 95%
    • 0 - se paciente teve custos mdicos at o percentil de 95%.

Avaliação da performance do modelo¶

Nesta etapa, sero investigados quatro algoritmos diferentes, sendo determinado posteriormente qual deles melhor para a modelagem dos dados. Trs destes algoritmos sero algoritmos de aprendizado supervisionado e o quarto algoritmo ser o naive predictor.

Métricas utilizadas¶

Estimativas indicam que 10% dos pacientes portadores de doenas crnicas representam 2/3 dos custos totais com sade no Brasil. Desta forma, o objetivo deste projeto ser predizer com acurcia quais indivduos possuem maior risco de ser um paciente de alto custo (H_COST). Usar a acurcia como uma mtrica para avaliar a performance de um modelo um parmetro adequado. Assim, a habilidade do modelo em predizer com preciso aqueles que possuem alto custo mais importante do que realizar o recall destes indivduos. Tambm ser utilizada a frmula F-beta score como uma mtrica que considera ambos: precision e recall.

In [17]:
# Calculo do accuracy, precision e recall
accuracy = n_maior_p95 / n_registros
recall = n_maior_p95 / (n_maior_p95+0)
precision = n_maior_p95 / (n_maior_p95+n_ate_p95)

#Clculo do F-score utilizando a frmula para o beta = 0.5 e os valores corretos de precision e recall.
fscore = (1+0.5*0.5)*(recall*precision)/((0.5*0.5 *precision)+recall)

Implementação - Criação do Pipeline de Treinamento e Predição¶

Nesta etapa, sero criados os pipelines de treinamento e predio para treinar os modelos utilizando vrios tamanhos de conjuntos de dados para treinamento, alm de performar predies nos dados de teste.

#Importao do fbeta_score e accuracy_score do sklearn.metrics

from sklearn.metrics import fbeta_score
from sklearn.metrics import accuracy_score

def train_predict (learner, sample_size, X_train, y_train, X_test, y_test): 
   
    results = {}
    
    start = time() 
    learner = learner.fit(X_train[:sample_size],y_train[:sample_size])
    end = time() 
    
    #Calculando o tempo de treinamento
    results['train_time'] = end - start
        
    # Predies do set de treinamento (X_test)
    # Predies das primeiros 300 amostras de treinamento 
    start = time() 
    predictions_test = learner.predict (X_test)
    predictions_train = learner.predict (X_train[:300])
    end = time() 
    
    # Clculo do tempo total de predio
    results['pred_time'] = end - start
            
    # Clculo do accuracy das primeiros 300 amostras de treinamento y_train[:300]
    results['acc_train'] = accuracy_score (y_true=y_train[:300],y_pred=predictions_train)
        
    # Clculo do accuracy no set de testes 
    results['acc_test'] = accuracy_score (y_true=y_test,y_pred=predictions_test)
    
    # Clculo do F-score das primeiros 300 amostras de treinamento usando fbeta_score()
    results['f_train'] = fbeta_score (y_true=y_train[:300],y_pred=predictions_train,beta=0.5)
        
    # Clculo do F-score no set de testes y_test
    results['f_test'] = fbeta_score (y_true=y_test,y_pred=predictions_test,beta=0.5)
       
    # Print dos Resultados
    print ("{} treinado em {} amostras.".format(learner.__class__.__name__, sample_size))
        
    # Resultados
    return results
